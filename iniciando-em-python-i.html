Iniciando em Python - Preparando o ambiente

Qui 31 maio 2018

Python is an easy to learn, powerful programming language.

The Python Tutorial

Inicio aqui uma série de postagens que reunirão minhas notas de estudos em Python.

Utilizarei o livro Python Crash Course como base e inicialmente, passarei por todos os capítulos da parte I.

Pretendo abordar todos os tópicos de forma bem sucinta, registrando somente minhas notas rápidas e o desenvolvimento de alguns exercícios.

Para este post em específico, separei alguns passos necessários para a preparação de um ambiente de desenvolvimento Python, de forma que fique isolado do Python do sistema, além de oferecer a possibilidade de ter uma versão diferente para cada projeto distinto.

Ferramentas necessárias

  • Linux, nesse caso Ubuntu 16.04

Verificando a versão do Ubuntu/Linux:

$ cat /etc/lsb-release 
DISTRIB_ID=Ubuntu
DISTRIB_RELEASE=16.04
DISTRIB_CODENAME=xenial
DISTRIB_DESCRIPTION="Ubuntu 16.04.3 LTS"
  • Python 3.6

Verificando a versão padrão do Python

$ python -V 
Python 2.7.12
$ python3 -V
Python 3.5.2
$

Antes de tudo, preparar o ambiente

O Ubuntu já vem com as versões 2 e 3 do Python. Mas um requisito importante antes de começar a instalar coisas no seu Python é criar um ambiente virtual.

Esse procedimento irá isolar sua versão do Python, sendo possível criar um ambiente para cada projeto e além disso, preservar a versão original do sistema.

Instalando o Pyenv

No terminal execute os seguintes comandos:

$ sudo apt-get update
$ sudo apt-get install -y make build-essential libssl-dev zlib1g-dev libbz2-dev libreadline-dev libsqlite3-dev wget curl llvm libncurses5-dev git
$ curl -L https://raw.githubusercontent.com/yyuu/pyenv-installer/master/bin/pyenv-installer | bash
$ pyenv update

Ao final da instalação, adicionar as seguintes linhas em seu arquivo ~/.bashrc.

export PATH="$HOME/.pyenv/bin:$PATH"
eval "$(pyenv init -)"
eval "$(pyenv virtualenv-init -)"

Executando o comand pyenv será possível ver os comando disponíveis. Alguns deles serão utilizados com mais frequência:

$ pyenv
pyenv 1.1.5

Instalando o Python no Pyenv

Verifique as versões do Python instaladas no Pyenv:

$ pyenv versions
  system

Somente a versão system será exibida, indicando que somente a versão do sistema está instalada.

Para instalar novas versões digite:

$ python install 3.6

Onde 3.6 é a versão desejada. Para ver todas as versões disponíveis digite:

$ python install -l

Ativando a versão do Python instalada

E para ativar a versão desejada, digite:

$ pyenv global 3.6.0

Nesse momento a versão 3.6.0 é a padrão. Para confirmar, consulte a versão do Python que está ativada:

$ python -V
Python 3.6.0

Criando o ambiente virtual

Agora, para isolar uma versão específica do Python, escolha um diretório onde será alocado o projeto. Dentro desse local, execute os seguinte comandos:

$ python -m venv .py3

O comando acima, basicamente realiza as seguinte operações:

  • python -m: sintaxe para rodar um script com Python;
  • venv: o nome do script do pyenv responsável por criar ambientes virtuais;
  • .py3: nome do meu ambiente virtual. Esse nome poderia ser outro.

Após isso, será criado um diretório, que nesse caso é oculto, e dentro dele existirá um versão do Python 3.6.0 isolada.

Nesse ambiente será possível instalar novos pacotes sem prejudicar a versão global do pyenv e muito menos a versão original do sistema.

Ativando o ambiente virtual

Para ativar o ambiente virtual, execute os seguintes comandos:

$ source .py3/bin/activate

Um prefixo aparecerá no bash, indicando que o ambiente está ativo.

(.py3) $

Desativando o ambiente virtual

Para desativar o ambiente virtual, execute o seguinte comando.

(.py3) $ deactivate
$

E agora?

Agora é só definir um diretório, criar um ambiente virtual, ativá-lo e codar.
